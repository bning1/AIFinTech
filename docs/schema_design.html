AIFinTech Trading Database Schema

Purpose: store capital markets instrument definitions, trades/positions, portfolios, and CCR counterparty info.

Entities

  • party: internal desks & external counterparties (CCR input)
  • portfolio: portfolio/desk containers
  • instrument: master instrument row for all asset classes
  • instrument_bond, instrument_equity: subtype extensions
  • trade: transaction record linking instrument/portfolio/counterparty
  • trade_leg_swap: swap leg details (for IR/CCS swaps/TRS-like legs)
  • trade_option_details: option-specific fields (swaption, cap/floor, options)
  • margin_agreement: minimal margin/CSA record for CCR extension

How to apply

createdb aifintech_trading
psql -d aifintech_trading -c "CREATE EXTENSION IF NOT EXISTS pgcrypto;"
psql -d aifintech_trading -f db_schema.sql

DDL location

The canonical DDL is stored in db_schema.sql.
